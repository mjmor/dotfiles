# host-setup

Infrastructure bootstrap for the Mac Mini M4 agent host.

## What this sets up

| Component | Where | Purpose |
|---|---|---|
| Colima (Apple VZ + Rosetta) | bare metal | Docker container runtime |
| `agents-net` Docker bridge | inside Colima VM | shared network for agent containers + minikube |
| Minikube | inside Colima VM | local Kubernetes cluster (docker driver) |
| Ollama | bare metal | LLM inference server, bound to `0.0.0.0:11434` |

Agent containers reach Ollama via `host.docker.internal:11434`. They reach the
Kubernetes API via `artifacts/kubeconfig-for-containers` (server URL rewritten
from `127.0.0.1` to minikube's `agents-net` IP).

## Prerequisites

| Requirement | How to check |
|---|---|
| macOS on Apple Silicon | `uname -m` → `arm64` |
| Homebrew | `brew --version` |
| Tailscale authenticated | `tailscale status` |

Bootstrap installs everything else (colima, docker, kubectl, minikube, ollama) via Homebrew.

## Run order

### 1. Tear down prior experiments (optional but recommended on a dirty machine)

```sh
cd host-setup
./pre-cleanup.sh
```

Removes: Lume VMs, launchd agents with `claude-`/`goose-`/`agentgateway-` labels,
standalone agentgateway/Goose binaries, Docker Desktop, all minikube profiles,
and old `agents-net`/`minikube` Docker networks.

### 2. Configure

Edit `config.env` to adjust resource limits or model lists before running bootstrap.

### 3. Bootstrap

```sh
./bootstrap.sh
```

Idempotent — safe to re-run. Prints a full verification summary at the end.

## Configuration reference

| Variable | Default | Description |
|---|---|---|
| `COLIMA_CPUS` | `6` | vCPUs for Colima VM |
| `COLIMA_MEMORY` | `8` | RAM for Colima VM (GiB) |
| `COLIMA_DISK` | `100` | Disk for Colima VM (GiB) |
| `MINIKUBE_MEMORY` | `4096` | RAM for minikube (MiB) |
| `MINIKUBE_CPUS` | `2` | CPUs for minikube |
| `MINIKUBE_ADDONS` | see below | Space-separated addon names |
| `OLLAMA_MODELS_LOCAL` | `nemotron-3-nano:4b gemma4:e4b` | Models pulled at bootstrap |
| `OLLAMA_MODELS_CLOUD` | `gemma4:31b-cloud` | Larger models pulled at bootstrap |

Default addons: `ingress metrics-server storage-provisioner registry`

## Verification

Bootstrap prints verification output automatically. To re-verify manually:

```sh
colima status
docker network inspect agents-net
minikube status
kubectl get nodes  # host-side; uses ~/.kube/config with 127.0.0.1
ollama list
```

To confirm Ollama is reachable from inside a container on `agents-net`:

```sh
docker run --rm --network=agents-net curlimages/curl:latest \
  curl -s http://host.docker.internal:11434/
```

## Teardown

```sh
./teardown.sh
```

Stops Ollama launchd service, deletes minikube, removes `agents-net`, stops Colima,
and cleans `artifacts/kubeconfig-for-containers`.

## Artifacts

`artifacts/kubeconfig-for-containers` — generated by bootstrap; **not committed to git**.
Mount this into agent containers at their expected kubeconfig path. The server URL
is `https://<minikube-agents-net-ip>:8443` — reachable from containers on `agents-net`
but **not** from the Mac host (use `~/.kube/config` for host-side kubectl).
